\section{Methodology}
\label{sec:method}

\subsection{Dataset}
\label{subsec:dataset}

We used the RQ1 subset of the \emph{ImproBR Replication Package}, publicly
available on Figshare~\cite{akyol2026improbr,improbr2026dataset}. This subset
contains 139 real-world bug reports collected from Mojira, the official issue
tracker for Minecraft, in \emph{two forms}:

\begin{itemize}
  \item \textbf{Raw}: the 139 reports in their original, as-submitted form;
  \item \textbf{Improved}: the same 139 reports after the ImproBR LLM pipeline
  rewrote each one to be more complete (clarifying or adding steps, observed
  behavior, expected behavior, and environment).
\end{itemize}

We treat the two forms as \emph{co-equal conditions} and run the identical
pipeline on each, because our research question concerns the model's agreement
with humans on both original and AI-improved reports. Each of the 278 report
instances (139 issues $\times$ 2 forms) was independently evaluated by two human
annotators (Author~1 and Author~2) along three structured dimensions defined in
the replication package's rubric (\path{evaluation_metrics.yaml}):

\begin{itemize}
  \item \textbf{Steps to Reproduce (S2R)}: \emph{Executable} or
  \emph{Non-Executable}, with an irreproducibility category when applicable;
  \item \textbf{Observed Behavior (OB)}: presence (\emph{Present}/\emph{Not
  Present}) and, when present, sufficiency (\emph{Sufficient}/\emph{Insufficient});
  \item \textbf{Expected Behavior (EB)}: presence and, when present, accuracy
  (\emph{Accurate}/\emph{Inaccurate}).
\end{itemize}

\subsection{Deterministic Scoring Function}
\label{subsec:scoring}

To obtain a single ordinal ground-truth score per report, we applied a fixed,
deterministic scoring function to the structured labels of each annotator:

\begin{equation}
\text{score}(s2r, irr, ob, obl, eb, ebl) =
\begin{cases}
5 & \text{if } s2r=\text{Exec.}, ob=\text{Suff.}, eb=\text{Acc.}\\
4 & \text{if } s2r=\text{Exec.}, \text{otherwise}\\
1 & \text{if } irr=\text{``wrong information''}\\
3 & \text{if } ob \text{ and } eb \text{ both present}\\
2 & \text{if only one of } ob, eb \text{ present}\\
1 & \text{otherwise}
\end{cases}
\label{eq:score}
\end{equation}

The same function was applied, separately for each form (Raw and Improved), to
(i) the adjudicated consensus labels in \path{Final Results.csv}, which
reconciles the two independent annotators (Author~1 and Author~2) and serves as
the ground truth for RQ1/SQ1--SQ3, and (ii) the structured labels extracted
from the LLM's JSON output, so that any difference between human and LLM scores
reflects a difference in how the report was judged rather than a difference in
scoring logic. For each form we also report the inter-annotator agreement
between Author~1 and Author~2's composite scores (before adjudication) as a
human baseline ($\kappa_{\text{control}}$, Section~\ref{subsec:stats}).
Applying the scoring function to the human ground truth confirms that ImproBR
substantially raises reproducibility: the share of reports scored as executable
($\geq 4$) rises from $28.8\%$ (40/139) on the Raw form to $67.6\%$ (94/139) on
the Improved form.

\subsection{LLM Configuration}
\label{subsec:llm}

We used \texttt{gpt-4o-mini-2024-07-18}~\cite{openai2024gpt4omini} with
\texttt{temperature}=0, a fixed \texttt{seed}, and JSON-mode structured
output, so that the model returns a label, quality assessment, and short
rationale for each of the S2R, OB, and EB dimensions. Following the
pre-registered protocol, we used a single fixed zero-shot prompt template,
anchored to the rubric in \path{evaluation_metrics.yaml}, for all 139 reports
of each form (one fixed template per form). The prompt supplies the report's
summary, description, and available metadata, and instructs the model to judge
only the report content, without access to ground-truth labels, annotator
identities, or prior predictions. For each form, a 26-report pilot phase
(Section~\ref{subsec:dataset}) confirmed the zero-shot template as sufficient
($\kappa$ between the LLM and the pilot ground truth of $0.567$ on Raw and
$0.576$ on Improved, both above the $0.50$ gate), so the pre-defined few-shot
fallback was not triggered. All reports were scored in a single pass with the
fixed prompt version, held identical between the pilot and the full run; we did
not revise the prompt after inspecting evaluation results.

\subsection{Statistical Analysis}
\label{subsec:stats}

All of the following analyses are run \emph{separately for each form} (Raw and
Improved), and our main comparison is between the two resulting sets of
numbers.

\textbf{RQ1/SQ1 (overall agreement).} For each form, we computed Cohen's
kappa~\cite{cohen1960coefficient} between the LLM's composite 1--5 score and
the human ground-truth score across all 139 reports, together with a
percentile bootstrap 95\% confidence interval (2{,}000 resamples with
replacement over reports). As a human baseline, we computed
$\kappa_{\text{control}}$ between Author~1 and Author~2's composite scores on
the same reports of that form.

\textbf{SQ2 (per-dimension agreement).} We re-scored each dimension
separately on a 1--3 scale (S2R: executable vs.\ not; OB: present-and-sufficient,
present-only, or absent; EB: present-and-accurate, present-only, or absent) and
computed Cohen's kappa independently for S2R, OB, and EB, applying the same
rescoring function to both human and LLM labels.

\textbf{SQ3 (disagreement analysis).} We identified reports where the LLM and
human composite scores differed and inspected the model's per-dimension
rationale text alongside the corresponding human labels to characterize
recurring disagreement patterns, following the four categories anticipated in
the pre-registered protocol (missing/ambiguous steps, implicit vs.\ explicit
expectations, unclear environment/version information, unstructured
description).

\textbf{Bias and sanity checks.} We ran a two-sided Wilcoxon signed-rank test
($\alpha=0.05$) on the paired (human, LLM) composite scores to test for a
systematic scoring shift, and, as a coarse sanity check, binarized both scores
at the pre-registered cut (score $\geq 4 \Rightarrow$ ``Reproducible'') and
report accuracy, precision, and recall for that binary task.

All statistics were computed with \texttt{scikit-learn} and \texttt{scipy} on
the released ground-truth and LLM-output files for each form
(\path{full_ground_truth_raw.csv} / \path{full_llm_output_raw.csv} and
\path{full_ground_truth_improved.csv} / \path{full_llm_output_improved.csv}).
The source code, prompt templates, and raw experimental outputs are
available at \url{https://github.com/LamSongToan/SWT301_SE1925_G6}.
